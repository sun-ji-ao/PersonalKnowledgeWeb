## 2. 当前项目定位

当前项目是一个纯静态个人知识网站，不需要后端服务。

核心形态：

```text
bypassAV-study-main Markdown 文档
    |
    v
scripts/import_markdown.mjs 自动导入
    |
    v
Astro Content Collection
    |
    v
静态文章页 / 分类页 / 标签页 / 首页信息流 / 搜索索引
    |
    v
GitHub / Cloudflare Pages / 自有域名
```

当前技术栈：

| 模块 | 当前实现 |
| --- | --- |
| 网站框架 | Astro（自定义页面，未使用 Starlight） |
| 内容来源 | 仓库内 `bypassAV-study-main` |
| Markdown 渲染 | Astro Content Collection |
| 代码高亮 | Astro / Shiki |
| 全文搜索 | Pagefind + 本地标题索引兜底 |
| 样式 | `src/styles/global.css` |
| SEO | `src/components/SeoHead.astro` |
| 草稿规则 | `content_rules.json` |
| 日期持久化 | `content_dates.json` |
| 部署方式 | 静态部署，推荐 Cloudflare Pages |
| 后端服务 | 当前不需要 |

## 3. 当前已实现功能

### 3.1 Markdown 自动导入

已实现。

相关文件：

```text
scripts/import_markdown.mjs
src/content.config.ts
```

功能说明：

- 从仓库内 `bypassAV-study-main` 读取全部 `.md` 文件。
- 自动复制到 `src/content/docs`。
- 自动补齐 frontmatter。
- 自动提取标题、分类、标签、排序。
- 支持两种文件名格式：`课时XX_标题.md` 与 `XX_标题.md`。
- 自动规范化部分不兼容的代码块语言。
- 自动重写 Markdown 和 HTML 中的本地相对图片路径。
- 自动读取 `content_rules.json`，按规则生成 `draft` 状态。
- 自动读取并更新 `content_dates.json`，区分历史文章与新增文章日期。
- 自动删除正文开头一级标题，避免文章页重复显示大标题。
- 自动为未编号的二级到四级 Markdown 标题补充 `1、`、`1.1、`、`1.1.1、` 形式编号。
- 已有编号的标题会保持原样，不会重复编号。
- 构建时自动执行导入。

当前构建脚本：

```json
"build": "npm run import && astro build && pagefind --site dist"
```

### 3.2 文章详情页

已实现。

相关文件：

```text

src/lib/content.ts
```

功能说明：

- 标题下方显示展示日期（历史文章为随机展示日期，新增文章为真实上传日期）。

- 文章目录位于左侧栏分类和标签信息下方。
- 正文标题在导入时统一补齐编号，文章目录读取同一份标题内容，保证正文和目录编号一致。
- 文章目录继续按标题层级缩进。
- 支持上一篇 / 下一篇。
- 支持阅读进度条。
- 支持代码块复制。
- 支持返回顶部。

- `npm run dev` 不会生成 Pagefind 索引，因此开发预览时显示的是兜底索引结果。

### 3.16 草稿和隐藏规则

已实现。

相关文件：

```text
content_rules.json
scripts/import_markdown.mjs
src/lib/content.ts
```

功能说明：

- `content_rules.json` 用于配置需要隐藏的目录、路径、分类、标题或标签。
- 导入脚本会根据规则写入 `draft: true`。
- `getPublishedDocs()` 会过滤草稿文章。
- 文章页、分类页、标签页、归档、RSS、sitemap 均基于公开文章生成。

当前默认规则：

```json
{
  "draftPathPrefixes": [],
  "draftPathIncludes": [],
  "draftCategories": [],
  "draftTitles": [],
  "draftTags": []
}
```

说明：默认不隐藏任何文章，只有明确写入规则后才会将文章标记为草稿。

### 3.19 展示日期系统

已实现。

相关文件：

```text
src/lib/content.ts
src/pages/docs/[...slug].astro
src/pages/archive/index.astro
src/pages/index.astro
src/pages/page/[page].astro
```

功能说明：

- 文章页、首页、归档页统一使用展示日期。
- 历史文章（`useRealDate: false`）：展示日期随机分布在 2021 年 11 月 1 日至 2025 年 10 月 31 日。
- 新增文章（`useRealDate: true`）：展示日期等于 frontmatter 中的 `date`（真实上传日期）。
- 分界规则：2026 年 5 月 1 日及之后首次导入的文章记为新增文章。
- 同专题内按 `order` 递增，相邻文章间隔不超过 30 天，单专题跨度不超过 90 天。
- 年份分布偏向 2022、2023、2024，并保证每年每季度至少 1 篇文章。
- 单月文章上限为 20 篇。

相关文件：

```text
content_dates.json
scripts/import_markdown.mjs
```

## 7. 内容更新流程

### 7.1 新增文章

推荐操作：

1. 把新的 `.md` 文件放入 `bypassAV-study-main` 对应目录。
2. 文件名使用 `课时XX_标题.md` 或 `XX_标题.md` 格式。
3. 执行：

```bash
npm run build
```

3. 本地预览确认页面。
4. 提交并推送：

```bash
git add .
git commit -m "Update knowledge base content"
git push origin main
```

### 7.2 修改文章

推荐操作：

1. 修改 `bypassAV-study-main` 中的原始 Markdown。
2. 不要直接长期修改 `src/content/docs`。
3. 执行：

```bash
npm run build
```

4. 确认无构建错误后推送。

原因：

- `src/content/docs` 是导入生成目录。
- 每次构建会重新生成。

### 7.3 删除文章

推荐操作：

1. 删除 `bypassAV-study-main` 中对应 `.md` 文件。
2. 执行：

```bash
npm run build
```

3. 确认文章页面已消失。
4. 提交并推送。

### 7.4 隐藏文章或目录

推荐操作：

1. 修改 `content_rules.json`。
2. 按路径、分类、标题或标签添加隐藏规则。
3. 执行：

```bash
npm run build
```

4. 确认对应文章不再出现在文章页、分类页、标签页、归档、RSS 和 sitemap 中。

示例：

```json
{
  "draftPathPrefixes": ["20_免杀专题/"],
  "draftPathIncludes": [],
  "draftCategories": [],
  "draftTitles": [],
  "draftTags": []
}
```

字段说明：

| 字段 | 用途 |
| --- | --- |
| `draftPathPrefixes` | 按原始 Markdown 相对路径前缀隐藏 |
| `draftPathIncludes` | 按原始 Markdown 相对路径片段隐藏 |
| `draftCategories` | 按自动生成的分类名隐藏 |
| `draftTitles` | 按自动生成的文章标题隐藏 |
| `draftTags` | 按自动推断的标签隐藏 |

注意：不要直接修改 `src/content/docs` 中的 frontmatter，因为该目录会在导入时重新生成。

## 11. 域名绑定操作

如果使用 Cloudflare Pages：

1. 在 Cloudflare Pages 项目中点击 Custom domains。
2. 添加你的域名，例如：

```text
www.yourdomain.com
```

3. 按 Cloudflare 提示添加 DNS 记录。
4. 设置环境变量：

```text
SITE_URL=https://www.yourdomain.com
```

5. 重新部署。

作用：

- RSS 链接会使用真实域名。
- sitemap 链接会使用真实域名。
- SEO 效果更稳定。
