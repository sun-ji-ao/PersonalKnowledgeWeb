# 1.5 部署上线操作手册

## 1. 文档说明

本文档说明如何将个人知识网站部署到 **GitHub Pages**，并将阿里云域名 **`sunjiao.me`** 绑定到线上站点。

当前状态：**部署已完成**，线上站点可访问。本文档同时作为首次配置记录与日常维护 / 排障手册。

适用仓库：

```text
https://github.com/sun-ji-ao/PersonalKnowledgeWeb
```

适用技术栈：

```text
Astro 静态站点 + Mermaid 图表 + Pagefind 搜索索引 + GitHub Actions 自动部署
```

相关文件：

```text
.github/workflows/deploy.yml
astro.config.mjs
package.json
```

预期结果：

- 无报错退出。

### 3.2 确认 GitHub 仓库权限

需要拥有 `PersonalKnowledgeWeb` 仓库的以下权限：

- 推送代码到 `main` 分支。

- 域名 DNS 解析服务可用（默认使用阿里云云解析 DNS）。

## 4. GitHub Pages 配置

本项目采用 **GitHub Actions 部署 GitHub Pages**，不使用手动上传 `dist/` 的方式。

说明：以下配置步骤已全部完成；若重建环境或排查问题，可按相同流程复核。

### 4.1 推送自动部署工作流

仓库中已包含工作流文件：

```text
.github/workflows/deploy.yml
```

工作流作用：

1. 监听 `main` 分支推送。
2. 执行 `npm ci` 安装依赖。
3. 执行 `npm run build` 构建站点。
4. 将 `dist/` 发布到 GitHub Pages。

关键环境变量：

```text
SITE_URL=https://sunjiao.me
```

如果本地还没有该文件，请创建 `.github/workflows/deploy.yml`，内容如下：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Build site
        run: npm run build
        env:
          SITE_URL: https://sunjiao.me

      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 4.2 启用 GitHub Pages

1. 打开仓库页面：
   `https://github.com/sun-ji-ao/PersonalKnowledgeWeb`
2. 点击顶部 **Settings**。
3. 左侧选择 **Pages**。
4. 在 **Build and deployment** 区域设置：
   - **Source** 选择 `GitHub Actions`
5. 保存后不要选择 `Deploy from a branch`。

说明：

- 本项目是 Astro 静态站点，必须通过 Actions 先执行 `npm run build`。
- 不能直接让 GitHub Pages 从 `main` 分支读取源码目录。

### 4.3 首次触发部署

任选一种方式：

**方式 A：推送代码**

```bash
git add .
git commit -m "Add GitHub Pages deployment workflow"
git push origin main
```

**方式 B：手动运行工作流**

1. 打开仓库 **Actions** 页签。
2. 选择 `Deploy to GitHub Pages`。
3. 点击 **Run workflow**。
4. 选择 `main` 分支后执行。

### 4.4 检查部署结果

1. 进入仓库 **Actions**。
2. 打开最新的 `Deploy to GitHub Pages` 工作流。
3. 确认 `build` 和 `deploy` 两个 Job 都是绿色成功状态。
4. 回到 **Settings → Pages**。
5. 页面顶部会显示站点地址，初始可能类似：

```text
https://sun-ji-ao.github.io/PersonalKnowledgeWeb/
```

说明：

- 在绑定自定义域名前，先确认 Actions 部署成功。
- 绑定 `sunjiao.me` 后，正式访问地址会切换为自定义域名。

### 4.5 配置站点根地址

项目通过 `astro.config.mjs` 读取 `SITE_URL`：

```javascript
site: process.env.SITE_URL ?? 'https://sunjiao.me'
```

该变量影响：

- RSS 链接。
- Sitemap 链接。
- Canonical 地址。
- Open Graph URL。

线上构建时，工作流已固定传入：

```text
SITE_URL=https://sunjiao.me
```

本地如需模拟线上地址，可执行：

```bash
set SITE_URL=https://sunjiao.me
npm run build
```

PowerShell 示例：

```powershell
$env:SITE_URL = "https://sunjiao.me"
npm run build
```

## 5. 域名绑定与阿里云 DNS 配置

目标：让 `sunjiao.me` 和 `www.sunjiao.me` 都访问 GitHub Pages 上的本站。

### 5.1 GitHub 侧配置自定义域名

1. 打开仓库 **Settings → Pages**。
2. 找到 **Custom domain**。
3. 输入：

```text
sunjiao.me
```

4. 点击 **Save**。
5. 等待 GitHub 检测 DNS。
6. 检测通过后，勾选 **Enforce HTTPS**。

说明：

- 首次保存后，GitHub 会提示需要配置 DNS 记录。
- DNS 未生效前，状态可能显示 `DNS check unsuccessful`，属于正常现象。
- 一般等待 10 分钟到 2 小时，最长可能需要 24 小时。

### 5.2 阿里云 DNS 解析配置

登录阿里云：

```text
https://dns.console.aliyun.com/
```

操作路径：

```text
域名解析 DNS → 权威域名解析 → 选择 sunjiao.me → 解析设置 → 添加记录
```

#### 记录 1：根域名 A 记录

GitHub Pages 根域名需要配置 4 条 **A 记录**。

| 记录类型 | 主机记录 | 记录值 | TTL |
| --- | --- | --- | --- |
| A | `@` | `185.199.108.153` | 10 分钟 |
| A | `@` | `185.199.109.153` | 10 分钟 |
| A | `@` | `185.199.110.153` | 10 分钟 |
| A | `@` | `185.199.111.153` | 10 分钟 |

说明：

- 主机记录 `@` 表示根域名 `sunjiao.me`。
- 需要添加 4 条 A 记录，不要只添加 1 条。

#### 记录 2：www 子域名 CNAME

| 记录类型 | 主机记录 | 记录值 | TTL |
| --- | --- | --- | --- |
| CNAME | `www` | `sun-ji-ao.github.io` | 10 分钟 |

说明：

- 该记录用于访问 `https://www.sunjiao.me`。
- 记录值填写你的 GitHub Pages 默认域名，不带 `https://`。

#### 可选：删除冲突记录

如果阿里云里已有以下记录，请先删除或停用，避免冲突：

- `@` 的 CNAME 记录。
- `@` 指向其他服务器 IP 的 A 记录。
- `www` 指向其他平台的 CNAME 记录。

### 5.3 建议的最终 DNS 记录

完成后，`sunjiao.me` 建议至少包含以下记录：

| 主机记录 | 类型 | 记录值 |
| --- | --- | --- |
| `@` | A | `185.199.108.153` |
| `@` | A | `185.199.109.153` |
| `@` | A | `185.199.110.153` |
| `@` | A | `185.199.111.153` |
| `www` | CNAME | `sun-ji-ao.github.io` |

### 5.4 等待 DNS 生效

可用以下方式检查：

**Windows：**

```bash
nslookup sunjiao.me
nslookup www.sunjiao.me
```

**在线工具：**

- 阿里云解析诊断。
- `https://www.whatsmydns.net/`

预期结果：

- `sunjiao.me` 解析到 GitHub Pages 的 IP。
- `www.sunjiao.me` 解析到 `sun-ji-ao.github.io`。

### 5.5 启用 HTTPS

当 GitHub Pages 检测到 DNS 生效后：

1. 回到 **Settings → Pages**。
2. 确认 **Custom domain** 为 `sunjiao.me`。
3. 勾选 **Enforce HTTPS**。
4. 等待证书签发完成。

完成后访问：

```text
https://sunjiao.me
https://www.sunjiao.me
```

## 8. 常见问题

### 8.1 GitHub Pages 显示 404

可能原因：

- 还没有成功运行 `Deploy to GitHub Pages` 工作流。
- Pages 源配置成了 `Deploy from a branch`，而不是 `GitHub Actions`。
- 最新一次 Actions 构建失败。

处理：

1. 检查 **Actions** 是否成功。
2. 确认 **Settings → Pages → Source = GitHub Actions**。
3. 重新运行工作流。

### 8.2 自定义域名一直显示 DNS 检查失败

可能原因：

- 阿里云 A 记录未配全 4 条。
- 根域名 `@` 同时存在冲突的 CNAME。
- DNS 还未生效。

处理：

1. 按本文第 5.2 节重新核对记录。
2. 删除冲突记录。
3. 等待 10 分钟到 2 小时后重试。

### 8.3 网站能打开，但 RSS / Sitemap 还是旧域名

可能原因：

- 构建时未传入 `SITE_URL=https://sunjiao.me`。

处理：

1. 检查 `.github/workflows/deploy.yml` 是否设置了 `SITE_URL`。
2. 重新推送代码触发部署。

### 8.4 `www.sunjiao.me` 不能访问

可能原因：

- 未配置 `www` 的 CNAME 记录。

处理：

在阿里云添加：

```text
主机记录：www
记录类型：CNAME
记录值：sun-ji-ao.github.io
```

### 8.5 本地能 build，GitHub Actions 失败

优先检查：

- Node 版本是否为 22（Astro 当前要求 `>=22.12.0`）。
- 仓库是否包含 `bypassAV-study-main`。
- 是否误删 `content_dates.json`。
- 导入脚本是否报错。

## 9. 首次上线顺序（已完成，仅供重建参考）

如需在新环境重新上线，建议严格按以下顺序执行：

1. 本地执行 `npm run build`，确认构建通过。
2. 推送 `.github/workflows/deploy.yml` 到 GitHub。
3. 在仓库 Settings 中启用 **GitHub Actions** 类型的 Pages。
4. 触发首次部署并确认 Actions 成功。
5. 在 GitHub Pages 中填写自定义域名 `sunjiao.me`。
6. 在阿里云 DNS 添加 4 条 A 记录和 1 条 `www` CNAME。
7. 等待 DNS 生效。
8. 在 GitHub Pages 勾选 **Enforce HTTPS**。
9. 按第 6 节清单逐项验证。
