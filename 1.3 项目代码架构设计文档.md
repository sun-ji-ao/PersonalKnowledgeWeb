# 1.3 项目代码架构设计文档

## 1. 文档目标

本文档根据 `1.1 知识网站项目方案计划书.md` 和 `1.2 网站示意图/1.2 网站示意图.svg` 的内容，描述个人知识网站当前实际采用的代码架构。

设计原则：

- 只覆盖现有文档中已经提出的功能。
- 保持结构简单，方便后续维护。
- 优先使用静态站点方案，降低部署成本。
- 不做复杂后端设计。

## 2. 是否需要后端服务

结论：当前阶段不需要后端服务。

原因：

- Markdown 文章可以在构建阶段转换为静态页面。
- 分类、标签、文章目录都可以通过 Markdown 元数据和目录结构生成。
- 站内全文搜索可以使用 Pagefind 在构建阶段生成搜索索引。
- 代码高亮可以由 Astro / Shiki 在构建阶段完成。
- 部署目标是 GitHub Pages，适合纯静态网站。

因此，本项目当前采用“静态站点生成”架构：

```text
bypassAV-study-main/*.md
    |
    v
scripts/import_markdown.mjs
    |
    v
src/content/docs/*.md
    |
    v
Astro 自定义页面构建（含 Mermaid 预处理）
    |
    v
静态 HTML / CSS / JS / 搜索索引
    |
    v
GitHub Pages（https://sunjiao.me）
```

只有在后续出现登录、评论、后台管理、私有文章权限、在线编辑等需求时，才需要考虑后端服务。当前版本已明确不实现文章上传页和在线编辑器。

## 3. 当前技术栈

当前实际技术栈：

| 模块 | 技术 |
| --- | --- |
| 网站框架 | Astro |
| 页面实现 | 自定义 Astro 页面与组件 |
| Markdown 渲染 | Astro Content Collection |
| 代码高亮 | Shiki |
| 图表渲染 | Mermaid（remark 插件 + 客户端渲染） |
| 站内搜索 | Pagefind + 开发环境标题索引兜底 |
| 样式定制 | `src/styles/global.css` |
| SEO | `src/components/SeoHead.astro` |
| 部署平台 | GitHub Pages（GitHub Actions 自动部署） |
| 自定义域名 | `sunjiao.me`（阿里云 DNS，已上线） |
| 内容来源 | 仓库内 `bypassAV-study-main` |

说明：

- 项目初期方案中提到 Starlight，但当前实现未接入 Starlight，而是使用自定义页面完成首页、知识库、文章页和归档页。
- 这样做更便于实现首页双栏博客流、展示日期系统和文章页左侧导航布局。

## 5. 核心模块设计

### 5.1 内容目录

路径：

```text
src/content/docs/
```

作用：

- 存放网站最终使用的 Markdown 文件。
- 保留 `bypassAV-study-main` 的一级目录结构。
- 每个一级目录对应网站的一个知识分类。

示例：

```text
src/content/docs/06_Windows内核安全/课时02_第一个驱动程序.md
```

### 5.2 Markdown 导入脚本

路径：

```text
scripts/import_markdown.mjs
```

作用：

- 从仓库内 `bypassAV-study-main` 读取 Markdown 文件。
- 按原始目录复制到 `src/content/docs/`。
- 自动补齐 frontmatter。
- 自动提取标题、分类、标签、排序（支持 `课时XX_` 与 `XX_` 两种文件名格式）。
- 自动修复本地图片路径。
- 自动删除正文开头一级标题，避免详情页重复显示大标题。
- 自动为未编号标题补齐层级编号。
- 根据 `content_rules.json` 写入草稿状态。
- 根据 `content_dates.json` 区分历史文章与新增文章日期，并写入 `useRealDate`。

处理流程：

```text
读取源目录
遍历所有 .md 文件
计算相对路径
复制到 src/content/docs
补齐 frontmatter
重写图片路径
规范化标题
```

### 5.3 内容查询与展示日期模块

路径：

```text
src/lib/content.ts
```

作用：

- 统一公开文章过滤逻辑。
- 提供 `getOrderedCategories()`，按专题目录序号排序分类。
- 提供 `formatDocListTitle()`，统一输出 `编号、文章名` 格式。
- 提供 `getCategoryUrl()`，生成 `/categories/专题名/` 链接。
- 提供分类、标签分组能力，专题内文章按 `order` 排序。
- 生成全站展示日期映射，历史文章随机分布，新增文章使用真实日期。
- 提供首页分页查询和归档分组能力。
- 提供 Markdown 正文摘要提取能力。

展示日期规则：

- 历史文章（`useRealDate: false`）：范围 2021-11-01 至 2025-10-31，同专题内按 `order` 递增。
- 新增文章（`useRealDate: true`）：直接使用 frontmatter 中的 `date` 作为展示日期。
- 分界时间：2026-05-01 及之后首次导入的文章记为新增文章。

### 5.4 首页组件

路径：

```text
src/components/HomeShell.astro
src/components/HomeArticleFeed.astro
src/pages/index.astro
src/pages/page/[page].astro
```

作用：

- `HomeShell` 负责左侧科技风导航栏。
- `HomeArticleFeed` 负责右侧文章列表和分页。
- 首页每页展示 10 篇文章，按展示日期倒序排列。

### 5.5 样式文件

路径：

```text
src/styles/global.css
```

作用：

- 实现深色极简科技风。
- 控制首页双栏布局、文章页布局、代码块、标签和分页样式。
- 支持明暗主题切换。
- 控制文章侧栏目录独立滚动（标题固定、列表可滚动、滚动条隐藏）。
- 控制 Mermaid 图表容器样式。

建议配色：

```text
背景色：#0B1020
卡片色：#111827
主文字：#E5E7EB
次文字：#9CA3AF
强调色：#38BDF8
辅助强调：#A78BFA
代码背景：#0F172A
```

### 5.6 组件目录

路径：

```text
src/components/
```

当前组件：

- `HomeShell.astro`：首页左侧导航壳。
- `HomeArticleFeed.astro`：首页文章流和分页。
- `SeoHead.astro`：统一 SEO 头部。

这些组件只负责展示，不负责数据存储。

### 5.7 Mermaid 图表渲染

路径：

```text
src/lib/remark-mermaid.mjs
src/lib/mermaid-client.ts
astro.config.mjs
src/pages/docs/[...slug].astro
```

作用：

- remark 插件在构建时将 ` ```mermaid ` 代码块转换为 `<div class="mermaid">`。
- 文章页客户端脚本调用 `mermaid.run()` 渲染 SVG。
- 若仍残留 `language-mermaid` 代码块，客户端会自动提升为图表容器再渲染。
- 主题切换时可根据 `data-theme` 重新渲染图表。

依赖：

```text
mermaid
unist-util-visit
@astrojs/markdown-remark
```

## 6. 页面结构设计

### 6.1 首页

功能：

- 左侧导航栏：品牌、搜索、分类专题、标签、归档、知识库、RSS、主题切换、分类列表。
- 右侧文章流：标题（`编号、文章名`）、展示日期、可点击专题名、摘要。
- 每页 10 条，支持分页。

数据来源：构建时读取 Markdown 和展示日期映射。

### 6.2 文档文章页

功能：

- 显示文章标题（`编号、文章名`）、分类、标签、展示日期、正文。
- 左侧栏提供返回首页、返回知识库、可点击分类名、标签和文章目录。
- 「文章目录」标题单独固定；目录列表独立滚动，滚动条隐藏。
- 显示代码高亮。
- 支持 Mermaid 流程图 / 时序图等图表渲染。
- 显示上一篇 / 下一篇导航。
- 支持阅读进度条、代码复制和返回顶部。

### 6.3 分类页

功能：

- 按一级目录展示文章，分类顺序按目录前缀序号排列。
- 显示分类下的文章列表，标题格式为 `编号、文章名`。

### 6.4 标签页

功能：

- 展示所有标签。
- 点击标签后显示相关文章。

### 6.5 归档页

功能：

- 按展示日期归档文章。
- 按月分组展示。

### 6.6 搜索页

功能：

- 搜索文章标题、正文、分类和标签。
- 构建环境使用 Pagefind。
- 开发环境使用内置标题索引兜底。

## 9. 配置文件设计

### 9.1 package.json

当前脚本：

```json
{
  "scripts": {
    "dev": "astro dev",
    "import": "node scripts/import_markdown.mjs",
    "build": "npm run import && astro build && pagefind --site dist",
    "preview": "astro preview"
  }
}
```

### 9.2 astro.config.mjs

作用：

- 配置 Astro 站点地址（默认 `https://sunjiao.me`）。
- 配置 Markdown / Shiki 代码高亮。
- 注册 `remarkMermaid` 插件，处理 Mermaid 代码块。

### 9.3 content_rules.json

作用：

- 控制哪些文章在导入时标记为 `draft: true`。
- 影响公开页面、RSS 和 sitemap 输出范围。

### 9.4 content_dates.json

作用：

- 持久化每篇文章的 `date`、`updated` 和 `useRealDate` 状态。
- 首次导入时自动将现有文章登记为历史文章。
- 后续新增的 Markdown 文件自动记录真实上传日期。
- 必须纳入版本控制，避免换环境后历史文章被误判为新文章。

### 9.5 部署工作流

路径：

```text
.github/workflows/deploy.yml
```

作用：

- 监听 `main` 分支推送。
- 自动执行 `npm ci` 和 `npm run build`。
- 将 `dist/` 发布到 GitHub Pages。
- 构建时注入 `SITE_URL=https://sunjiao.me`。

## 11. 不设计的内容

当前版本不设计以下功能：

- 用户登录。
- 评论系统。
- 后台管理系统。
- 数据库。
- 文章在线编辑。

这些功能如果以后需要，可以单独扩展，但不属于当前版本。

## 12. 简化后的最终架构

本项目最终可以理解为：

```text
一个 Astro 静态网站
    +
一个 Markdown 内容目录
    +
一个导入与规范化脚本
    +
一个内容查询与展示日期模块
    +
一套自定义页面与组件
    +
一个自定义 CSS 主题
    +
一个静态搜索索引
```

这套架构简单、成本低、部署方便，适合当前个人知识网站和技术博客的需求。
