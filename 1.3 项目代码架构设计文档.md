# 1.3 项目代码架构设计文档

## 1. 文档目标

本文档根据 `知识网站项目方案计划书.md` 和 `1.2 网站示意图/1.2 网站示意图.svg` 的内容，设计个人知识网站的代码架构。

设计原则：

- 只覆盖现有文档中已经提出的功能。
- 保持结构简单，方便后续维护。
- 优先使用静态站点方案，降低部署成本。
- 不做复杂后端设计。

## 2. 是否需要后端服务

结论：当前阶段不需要后端服务。

原因：

- Markdown 文章可以在构建阶段转换为静态页面。
- 分类、标签、文章目录都可以通过 Markdown 元数据和目录结构生成。
- 站内全文搜索可以使用 Pagefind 在构建阶段生成搜索索引。
- 代码高亮可以由 Astro / Starlight / Shiki 在构建阶段完成。
- 部署目标是 Cloudflare Pages 或 GitHub Pages，适合纯静态网站。

因此，本项目当前采用“静态站点生成”架构：

```text
Markdown 文件
    |
    v
导入与规范化脚本
    |
    v
Astro / Starlight 构建
    |
    v
静态 HTML / CSS / JS / 搜索索引
    |
    v
Cloudflare Pages / GitHub Pages
```

只有在后续出现登录、评论、后台管理、私有文章权限、在线编辑等需求时，才需要考虑后端服务。当前文档中没有这些功能，所以不设计后端。

## 3. 技术栈

推荐代码技术栈：

| 模块 | 技术 |
| --- | --- |
| 网站框架 | Astro |
| 文档主题 | Starlight |
| Markdown 渲染 | Astro Content Collection / Starlight Docs |
| 代码高亮 | Shiki |
| 站内搜索 | Pagefind |
| 样式定制 | CSS |
| 部署平台 | Cloudflare Pages 或 GitHub Pages |
| 内容来源 | 本地 Markdown 文件 |

## 5. 核心模块设计

### 5.1 内容目录

路径：

```text
src/content/docs/
```

作用：

- 存放网站最终使用的 Markdown 文件。
- 保留 `bypassAV-study-main` 的一级目录结构。
- 每个一级目录对应网站的一个知识分类。

示例：

```text
src/content/docs/06_Windows内核安全/课时02_第一个驱动程序.md
```

### 5.2 Markdown 导入脚本

路径：

```text
scripts/import_markdown.ts
```

作用：

- 从 `C:\Users\17536\Desktop\bypassAV-study-main` 读取 Markdown 文件。
- 按原始目录复制到 `src/content/docs/`。
- 保留原有层级关系。

处理流程：

```text
读取源目录
遍历所有 .md 文件
计算相对路径
复制到 src/content/docs
```

### 5.3 Frontmatter 规范化脚本

路径：

```text
scripts/normalize_frontmatter.ts
```

作用：

- 为 Markdown 文件补齐基础元数据。
- 从目录名生成分类。
- 从文件名生成标题和排序。
- 为标签体系预留字段。

建议文章元数据：

```yaml
---
title: 第一个驱动程序
description: Windows 内核驱动入门实验记录
category: Windows 内核安全
tags:
  - Windows
  - Kernel
  - Driver
date: 2026-07-09
updated: 2026-07-09
draft: false
order: 2
---
```

### 5.4 侧边栏生成脚本

路径：

```text
scripts/generate_sidebar.ts
```

作用：

- 根据 `src/content/docs/` 下的目录生成左侧分类导航。
- 根据文件名中的课时编号排序。
- 输出给 Starlight 使用的 sidebar 配置。

对应 UI：

- SVG 示意图中的左侧“知识分类”区域。

### 5.5 样式文件

路径：

```text
src/styles/custom.css
```

作用：

- 定制深色极简科技风。
- 控制背景色、卡片、边框、文字、代码块和标签样式。

建议配色：

```text
背景色：#0B1020
卡片色：#111827
主文字：#E5E7EB
次文字：#9CA3AF
强调色：#38BDF8
辅助强调：#A78BFA
代码背景：#0F172A
```

### 5.6 组件目录

路径：

```text
src/components/
```

当前只保留简单组件，不做复杂设计。

可包含：

- 标签展示组件。
- 文章卡片组件。
- 首页精选文章组件。

这些组件只负责展示，不负责数据存储。

## 6. 页面结构设计

根据示意图和计划书，网站页面分为以下几类。

### 6.1 首页

功能：

- 展示站点名称。
- 展示搜索入口。
- 展示精选文章。
- 展示最近更新。
- 展示分类入口。

不需要后端，数据来自构建时读取的 Markdown。

### 6.2 文档文章页

功能：

- 显示文章标题。
- 显示分类和标签。
- 渲染 Markdown 正文。
- 显示代码高亮。
- 显示右侧文章目录。
- 显示上一篇 / 下一篇导航。

不需要后端，页面由 Astro 构建生成。

### 6.3 分类页

功能：

- 按一级目录展示文章。
- 显示分类下的文章列表。

不需要后端，分类来自 Markdown 目录和 frontmatter。

### 6.4 标签页

功能：

- 展示所有标签。
- 点击标签后显示相关文章。

不需要后端，标签来自 Markdown frontmatter。

### 6.5 归档页

功能：

- 按时间展示文章。
- 方便后续博客化浏览。

不需要后端，时间来自 Markdown frontmatter。

### 6.6 搜索页或搜索弹窗

功能：

- 搜索文章标题、正文、标签和代码片段。

实现方式：

- 使用 Pagefind 生成搜索索引。
- 搜索在浏览器本地完成。

不需要后端搜索服务。

## 7. 数据流设计

```text
bypassAV-study-main/*.md
    |
    v
scripts/import_markdown.ts
    |
    v
src/content/docs/*.md
    |
    v
scripts/normalize_frontmatter.ts
    |
    v
Astro / Starlight
    |
    v
静态页面 + 文章目录 + 代码高亮
    |
    v
Pagefind 搜索索引
    |
    v
Cloudflare Pages / GitHub Pages
```

用户访问网站时的数据流：

```text
浏览器请求页面
    |
    v
静态 HTML 页面返回
    |
    v
浏览器加载 CSS / JS / 搜索索引
    |
    v
用户阅读文章或执行站内搜索
```

## 8. 构建与部署流程

### 8.1 本地开发流程

```text
编辑 Markdown
    |
运行导入脚本
    |
运行本地开发服务
    |
检查页面效果
```

建议命令：

```bash
npm run import
npm run dev
```

### 8.2 构建流程

```text
导入 Markdown
    |
生成分类和侧边栏
    |
Astro 构建静态页面
    |
Pagefind 生成搜索索引
    |
输出 dist 目录
```

建议命令：

```bash
npm run build
```

### 8.3 部署流程

```text
推送代码到 GitHub
    |
Cloudflare Pages 自动构建
    |
发布 dist 静态文件
    |
通过自有域名访问
```

## 9. 配置文件设计

### 9.1 package.json

作用：

- 管理项目依赖。
- 定义开发、导入、构建命令。

建议脚本：

```json
{
  "scripts": {
    "dev": "astro dev",
    "import": "tsx scripts/import_markdown.ts",
    "build": "npm run import && astro build && pagefind --site dist",
    "preview": "astro preview"
  }
}
```

### 9.2 astro.config.mjs

作用：

- 配置 Astro。
- 启用 Starlight。
- 配置站点标题、导航、侧边栏、搜索入口和样式。

### 9.3 src/styles/custom.css

作用：

- 实现示意图中的深色科技风。
- 调整导航栏、文章区、代码块和标签样式。

- 用户登录。
- 评论系统。
- 后台管理系统。
- 数据库。
- 文章在线编辑。
- 付费订阅。

- 权限系统。

这些功能如果以后需要，可以单独扩展，但不属于当前版本。

## 12. 简化后的最终架构

本项目最终可以理解为：

```text
一个 Astro 静态网站
    +
一个 Markdown 内容目录
    +
几个内容处理脚本
    +
一个自定义 CSS 主题
    +
一个静态搜索索引
```

这套架构简单、成本低、部署方便，适合当前个人知识网站和技术博客的需求。

## 13. 当前阶段建议实现顺序

1. 初始化 Astro + Starlight 项目。
2. 建立 `src/content/docs/` 内容目录。
3. 编写 Markdown 导入脚本。
4. 编写 frontmatter 规范化逻辑。
5. 生成侧边栏分类导航。
6. 配置 Shiki 代码高亮。
7. 添加 `custom.css` 实现极简科技风。
8. 接入 Pagefind 搜索。
9. 构建并部署到 Cloudflare Pages。

以上步骤完成后，即可满足当前文档中描述的主要功能。
